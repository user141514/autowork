# Reference Research

Status: pending.

本文件保留给 Phase 0/后续调研使用。当前用户要求先进入 MVP 项目骨架和 Phase 1，因此这里不伪造外部项目调研结论。

后续调研应覆盖：

- ChatOps: Slack / Discord / 企业微信 / 微信触发工作流的权限、命令解析和反馈模式。
- Coding Agent: Aider、OpenHands、SWE-agent、Claude Code 类项目的任务输入、diff、测试和提交方式。
- Workflow / Orchestrator: Temporal、LangGraph、Prefect、Dagster、AutoGen workflow 的状态机、重试、恢复和日志模式。
- GitOps / PR 自动化: branch、commit、PR 创建和状态回写模式。
- 桌面 Agent / 本地 Agent: 本地任务、文件系统、命令执行、日志和权限控制。

调研输出必须围绕“哪些设计适合本系统”展开，而不是简单堆项目列表。
